üìÑ Overview

VoidScan is an open-source minimalistic port scanner designed for cybersecurity professionals and pentesters who require stealthy and efficient network scans. Written in Rust for performance and safety, it focuses on minimizing detection by target systems while delivering reliable scan results.

Initiated in 2023 by the VoidLab team, VoidScan was born out of the need for a lightweight yet powerful tool that avoids the common pitfalls of noisy scanning utilities. It supports both TCP and UDP scanning modes and provides detailed logs for forensic analysis.

The project follows a strict no telemetry, no data collection policy, respecting user privacy and operational security.

‚ú® Features

  • Stealth TCP and UDP scanning modes with customizable timing.
  • Advanced thread pool to optimize scan speed without overwhelming networks.
  • Detailed logging in logs/trace.log with session metadata.
  • Output formats: JSON, XML, CSV for integration in SIEM tools.
  • Lightweight footprint & zero dependencies outside Rust standard libraries.
  • Extensible plugin system planned for future versions.
  • Supports IPv4 and IPv6 targets.
  • Minimal false positives with adaptive timeout strategies.

You can also download precompiled binaries from the Releases page.

üß™ Usage

Basic command to scan all TCP ports on an IP:

./voidscan -t 10.0.0.1 -p 1-65535

Scan UDP ports with -u flag:

./voidscan -t 10.0.0.1 -p 53-53 -u

Logs are saved in logs/trace.log. You can change output format with -o json, -o xml, or -o csv.

Run ./voidscan --help for full options.

The logs/trace.log file contains detailed scan sessions, including timestamps, open ports, and user tags.
The log format is plain text for easy parsing, with entries such as:

Timestamp: 2025-07-15T14:35:02Z
User: hack_the_void
Open ports: 22, 80, 443
GPS coords: 48.8566, 2.3522
Notes: Silent mode enabled

For further information, see the VoidScan Wiki.

Note: All development follows strict security guidelines and no personal data is collected.

Is VoidScan detectable by IDS/IPS?
